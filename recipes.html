This file contains all the reviews provided by the users for various restaurants in the Business.csv file. 

The business.csv file consists of the following fields:

  • business_id (String): unique ID for every restaurant in the dataset
  • name (String): The name of the restaurant
  • neighborhood (String): Neighborhood in which the restaurant is located
  • address (String): Address of the restaurant
  • city (String): City in which the restaurant is located
  • state (String): 2-character code denoting the state in which the restaurant is located
  • postal_code (Number): 5-digit zip code of the location
  • latitude (Double): Longitudinal location of the restaurant
  • longitude (Double): Latitudinal location of the restaurant
  • stars (Double): The overall star rating of the restaurant on Yelp
  • review_count (Integer): Number of reviews of the restaurant on Yelp
  • is_open (Binary): 0-1 denoting whether the restaurant is open or not
  • categories (String): The cuisines available at the restaurant

The dataset also consists of other information such as reviews, which we plan on utilizing for visualizations in the final submission. For the advanced visualizations and the predictions, we are using the reviews.csv dataset, which contain user reviews for each restaurant.

  • review_id (String): unique ID for every review in the dataset
  • user_id (String): unique ID for every user in the dataset
  • Business_id (String): unique ID for every restaurant in the dataset
  • stars (int) rating from 1 to 5
  • date (Date) Date of the review
  • text (String) Contents of the review

We performed a left join to filter the restaurants along with all their reviews.
We have also used the checkin.csv data file, which contains the working hours for each restaurant along with the footfall for all the 7 days of the week.

  • Business_id:(String): unique ID for every restaurant in the dataset
  • Hours of operation: (int) Specifies the hours for which the restaurant specified by the business id is open
  • footfall: (int) number of people visiting the restaurant

We performed a left join to get the names of the restaurant from the business id.
Data Cleaning using Wrangler. Steps taken to clean business.csv:

  • Keep records with valid state code in North America. This includes Canada and the United States. Other records are filtered out.
  • Trim quotes (""") from name and address fields.
  • Keep records with review_count >= 8
  • Drop postal_code and is_open columns since they are of no use in our analysis.

Steps taken to clean review.csv: Grouped all the reviews by business ids Performed a left join with business.csv to get all the restaurant names in the review.csv file For each restaurant, we further appended each review in a list.
